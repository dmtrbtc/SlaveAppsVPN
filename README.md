## ⭐ Почему SLAVE VPN

### 🇷🇺 Российские сервисы — всегда на российском IP
В режиме **«Обход»** банки, госуслуги и платёжные системы идут напрямую — даже если их CDN живёт за рубежом. Курируемый список РФ-сервисов (Сбер, ВТБ, Альфа, Т-Банк, ГПБ, НСПК/Мир, ЮMoney, Госуслуги, Kinopoisk/ivi/okko…) гарантирует им прямой маршрут. Никаких «сервис недоступен в вашем регионе».

### 🏦 Банковские приложения и VPN — больше не конфликт
Банки вроде ВТБ блокируют вход при любом активном VPN. Решается **в один тап**: «Приложения и VPN» → **«Банковские приложения — мимо VPN»** — и все банки идут напрямую, мимо туннеля, в любом режиме.

### 🎛️ Понятная маршрутизация
Режим — главный переключатель: **Полный VPN**, **Обход**, **Раздельный**, **Свой**. На дашборде — живой индикатор **«куда идёт трафик»**. В режиме «Свой» сценарии разделены на **Базу** (выбери один профиль) и **Дополнения** (комбинируй: AI, стриминг, игры, Ad Block, RuNet Freedom).

### 🔒 Anti-DPI под контролем
Выбор uTLS-отпечатка (маскировка Client Hello против поведенческого DPI ТСПУ) применяется **мгновенно**. Для REALITY-серверов автоматически подбирается совместимый отпечаток — стабильное соединение из коробки.

## ✨ Что умеет

- **Современные протоколы:** VLESS + **Reality** + **XTLS-Vision** (в т.ч. постквантовое шифрование **ML-KEM-768 / X25519**), **Hysteria2**, **TUIC**, **Trojan**, **VMess**, **Shadowsocks**.
- **4 режима маршрутизации** + гибкие сценарии обхода (РФ-напрямую, AI, стриминг, игры, RuNet Freedom).
- **Раздельный туннель** — выбор приложений, которые идут через VPN или мимо него.
- **Личный кабинет** — баланс, подписки, устройства, продление и автопродление. Вход по **Email** или **Telegram**.
- **Несколько подписок** с авто-обновлением, дедупликацией и выбором сервера по пингу.
- **Автобалансер** — режим «Авто» выбирает быстрейший узел; либо ручной выбор сервера.
- **Защищённый DNS** — DoH-пул через туннель, fake-ip, anti-leak, политики по доменам.
- **Живая диагностика** — трафик, активные соединения, логи ядра в реальном времени, тесты соединения, self-test.
- **Авто-обновление** — встроенная проверка новых версий (стабильный и dev-канал).

## 🚀 Как пользоваться

1. **Установите** приложение (см. «Скачать»).
2. **Добавьте подписку** — вставьте **URL подписки** на онбординге (можно просто скопировать ссылку в буфер — приложение её подхватит) или войдите в **личный кабинет**.
3. **Выберите сервер** на дашборде (или включите режим **«Авто»**).
4. Нажмите большую кнопку **«Подключить»**. При первом запуске Android запросит разрешение на VPN.
5. На экране **«Маршрутизация»** выберите режим: **Обход** (рекомендуется для РФ), **Полный VPN**, **Раздельный** или **Свой**.

> Для работы нужна **активная подписка с узлами** на стороне панели. Без узлов список серверов будет пуст.

| Протокол | Транспорт | Примечание |
|---|---|---|
| VLESS + Reality | TCP / XTLS-Vision | + шифрование ML-KEM-768 / X25519 |
| Hysteria2 | QUIC / UDP | QUIC/UDP может ограничиваться некоторыми операторами/DPI |
| TUIC | QUIC / UDP | — |
| Trojan | TCP / TLS | — |
| VMess | TCP / WS / gRPC | — |
| Shadowsocks | TCP / UDP | — |

Подписка читается из форматов: **Clash YAML**, **sing-box JSON**, **base64 (vless://…)**, **v2rayN/Xray-массив** (откуда дополнительно восстанавливаются Hysteria2-узлы).

## 🏗️ Архитектура (кратко)

Монорепозиторий (pnpm workspaces). **Один React-рендерер** (Aurora design system) обслуживает обе платформы; вся бизнес-логика — в платформо-независимом `@slave-vpn/core` поверх адаптеров.

```
apps/
  windows/        Electron (main + preload + renderer) — десктоп
  android/        Capacitor-обёртка + нативный VpnService (Kotlin) + mihomo.aar
packages/
  core/           Платформо-независимое ядро: маршрутизация, DNS, кабинет (над адаптерами)
  config/         Парсеры подписок, генератор mihomo-конфига, компиляторы
  routing/        Движок правил маршрутизации (DSL → pipeline → Mihomo rules)
  dns/            DNS-профили и компилятор
  shared/         Общие типы и модели
```

- **Windows:** Electron-main запускает `mihomo.exe` как отдельный процесс, общается по Clash-API; рендерер ↔ main — через типизированный IPC; `@slave-vpn/core` работает в main.
- **Android:** `mihomo` вкомпилирован в `.aar` (gomobile); нативный `VpnService` отдаёт TUN-дескриптор ядру; рендерер общается через мост `window.slaveVPN`; `@slave-vpn/core` работает в рендерере поверх Capacitor-адаптеров. Ядро инициализируется **только при подключении**.

**Ядро mihomo (Clash.Meta) — на обеих платформах**, поэтому поведение маршрутизации и DNS идентично.

## 🛠️ Сборка из исходников

Требуется: **Node 20+**, **pnpm**. Для Android дополнительно — JDK 21, Android SDK/NDK, Go + gomobile.

```bash
pnpm install

# Windows: установщик + portable (в apps/windows/release/)
pnpm --filter @slave-vpn/windows dist

# Проверка типов / сборка рендерера / тесты
pnpm --filter @slave-vpn/windows typecheck
pnpm --filter @slave-vpn/windows build
pnpm --filter @slave-vpn/config test
```

Android APK собирается в CI — `.github/workflows/android.yml`. Локальная сборка: см. [docs/ANDROID.md](./docs/ANDROID.md).

## 🔒 Приватность

Приложение **не собирает телеметрию**. Данные не отправляются никуда, кроме вашего VPN-сервера и сервера подписки. Весь трафик идёт через ваши собственные узлы.
